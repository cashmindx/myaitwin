📋 Prerequisites

Before you begin:

  • ✅ Yoco business account (South African business required)
  • ✅ Valid business registration documents
  • ✅ Bank account for settlements
  • ✅ SSL certificate for your domain
⚠️ Important: Yoco is primarily for South African businesses. International businesses should consider Stripe or PayPal.

Step 1: Create Yoco Account

1. Visit yoco.com

2. Click "Get Started" and choose your business type

3. Complete the application with:

  • Business registration details
  • Banking information
  • Identity verification
  • Business address

4. Wait for approval (usually 1-3 business days)

5. Access your dashboard and get API keys

Step 2: Environment Setup

Create a .env file in your project root:

# Yoco Configuration VITE_YOCO_PUBLIC_KEY=pk_test_your_public_key_here YOCO_SECRET_KEY=sk_test_your_secret_key_here YOCO_WEBHOOK_SECRET=whsec_your_webhook_secret_here # App Configuration VITE_APP_URL=https://your-domain.com VITE_API_URL=https://your-api.com
Security Note: Never expose secret keys in frontend code. Only use public keys in React components.

Step 3: Install Dependencies

# Install Yoco SDK and related packages npm install @yoco/yoco-sdk-web npm install axios uuid # For TypeScript support npm install @types/uuid --save-dev

Step 4: Frontend Integration

Update your payment component to use Yoco SDK:

// src/components/YocoPayment.tsx import { YocoSDK } from '@yoco/yoco-sdk-web'; const yoco = new YocoSDK({ publicKey: import.meta.env.VITE_YOCO_PUBLIC_KEY }); const handleYocoPayment = async (formData: PaymentFormData) => { try { // Create payment token const result = await yoco.showPopup({ amountInCents: parseFloat(price.replace('$', '')) * 100, currency: 'ZAR', // South African Rand name: 'My AI Twin', description: `${plan} Plan Subscription`, image: '/logo.png', locale: 'en', allowRememberMe: false, metadata: { plan: plan, userId: 'user_123', email: formData.email } }); if (result.error) { throw new Error(result.error.message); } // Send token to backend for processing const response = await fetch('/api/process-payment', { method: 'POST', headers: { 'Content-Type': 'application/json', }, body: JSON.stringify({ token: result.id, plan, price, email: formData.email }) }); if (response.ok) { setPaymentStep('success'); onSuccess(); } else { throw new Error('Payment processing failed'); } } catch (error) { console.error('Payment error:', error); alert('Payment failed: ' + error.message); } };

Step 5: Backend API Setup

Create a secure backend endpoint to process payments:

// api/process-payment.js (Node.js/Express example) const express = require('express'); const yoco = require('yoco'); const app = express(); // Initialize Yoco with secret key const yocoClient = yoco(process.env.YOCO_SECRET_KEY); app.post('/api/process-payment', async (req, res) => { try { const { token, plan, price, email } = req.body; // Validate input if (!token || !plan || !price || !email) { return res.status(400).json({ error: 'Missing required fields' }); } // Create charge const charge = await yocoClient.charges.create({ token: token, amountInCents: parseFloat(price.replace('$', '')) * 100, currency: 'ZAR', metadata: { plan: plan, email: email, timestamp: new Date().toISOString() } }); if (charge.status === 'successful') { // Update user subscription in database await updateUserSubscription(email, plan, charge.id); // Send confirmation email await sendConfirmationEmail(email, plan, charge.id); res.json({ success: true, chargeId: charge.id, plan: plan }); } else { res.status(400).json({ error: 'Payment failed', status: charge.status }); } } catch (error) { console.error('Payment processing error:', error); res.status(500).json({ error: 'Internal server error', message: error.message }); } }); async function updateUserSubscription(email, plan, chargeId) { // Update your database with subscription details // This is where you'd integrate with your user management system } async function sendConfirmationEmail(email, plan, chargeId) { // Send confirmation email to user // Integrate with your email service (SendGrid, Mailgun, etc.) }

Step 6: Webhook Configuration

Set up webhooks to handle payment confirmations and failures:

// api/yoco-webhook.js const crypto = require('crypto'); app.post('/api/yoco-webhook', express.raw({type: 'application/json'}), (req, res) => { const signature = req.headers['x-yoco-signature']; const payload = req.body; // Verify webhook signature const expectedSignature = crypto .createHmac('sha256', process.env.YOCO_WEBHOOK_SECRET) .update(payload) .digest('hex'); if (signature !== expectedSignature) { return res.status(400).send('Invalid signature'); } const event = JSON.parse(payload); switch (event.type) { case 'payment.succeeded': handleSuccessfulPayment(event.data); break; case 'payment.failed': handleFailedPayment(event.data); break; case 'payment.refunded': handleRefundedPayment(event.data); break; default: console.log('Unhandled event type:', event.type); } res.status(200).send('OK'); }); async function handleSuccessfulPayment(paymentData) { // Confirm subscription activation // Send welcome email // Log successful payment } async function handleFailedPayment(paymentData) { // Handle failed payment // Notify user // Log failure for analysis }

🛡️ Security Best Practices

  • Never store card details: Always use Yoco tokens
  • Validate webhooks: Always verify signatures
  • Use HTTPS: All payment endpoints must be secure
  • Log everything: Maintain audit trails
  • Handle errors gracefully: Provide clear user feedback
  • Rate limiting: Prevent abuse of payment endpoints
  • PCI compliance: Follow Yoco's security guidelines
